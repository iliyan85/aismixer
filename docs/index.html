AISMixer — merge, deduplicate & forward AIS NMEA streams

AISMixer — обединяване, дедупликация и форуърд на AIS NMEA потоци

Ingest multiple AIS receivers & apps, remove duplicates, normalize or add meta headers, and forward a clean unified NMEA stream to platforms like AISHUB / MarineTraffic / VesselFinder. Stations can use nmea_sproxy for an authenticated, encrypted, NAT‑friendly secure UDP path to AISMixer.

Приема потоци от няколко AIS приемника/софтуера, премахва дубликати, нормализира или добавя мета хедъри и изпраща чист обединен NMEA поток към платформи като AISHUB / MarineTraffic / VesselFinder. Станциите могат да използват nmea_sproxy за автентикирана, криптирана и удобна за NAT secure UDP връзка към AISMixer.

Key features

Основни възможности

🔁

Multiplex & Deduplicate

Merge many inputs, strip vendor meta, resolve duplicates, and keep the freshest message per MMSI/timestamp.

Обединява входовете, премахва vendor мета, открива дубликати и запазва най-актуалното съобщение по MMSI/време.

⚙️

Smart tags

Control c-tag policy and reassemble multipart via g-tags; handles glued lines and out-of-band headers.

Контрол на политиката за c-таг и сглобяване на multipart чрез g-таг; справя се със „залепени“ редове и външни хедъри.

🧭

Forwarding

Fan-out to multiple UDP/TCP targets; per-user isolation (vhost-like model) and per-station mapping.

Изпраща към множество UDP/TCP цели; изолация по потребител (vhost‑подобен модел) и мапване по станции.

🛡️

Implemented secure transport

Реализиран защитен транспорт

ECDSA station authentication, AES‑GCM encrypted data and ping/pong liveness, replay checks, and NOSESSION/peer_timeout recovery for NAT, CGNAT, and mobile clients.

ECDSA автентикация на станцията, AES‑GCM криптирани данни и ping/pong liveness, replay проверки и NOSESSION/peer_timeout възстановяване за NAT, CGNAT и мобилни клиенти.

📦

Queues & performance

Independent queues per port (plain/secure), zero‑copy friendly parsing, and TTL cache tuning.

Отделни опашки по порт (plain/secure), zero‑copy парсинг и оптимизации на TTL кеш.

🌍

IPv4 + IPv6

Full dual‑stack UDP for ingress/egress; station naming station_private.pem/station_public.pem.

Пълен dual‑stack UDP за вход/изход; именуване на ключовете station_private.pem/station_public.pem.

🔍

Observability

Planned metrics/health endpoints and test harness with known duplicate sets.

Планирани metrics/health endpoints и тестови сет с известни дубликати.

🚀

Cloud control

Future per‑user cloud panel to manage local stations remotely.

Бъдещ cloud панел за управление на локални станции по потребител.

Architecture

Архитектура

High‑level flow

Общ поток

  1. AIS receiver/app → local UDP → nmea_sproxy → encrypted SEC input → AISMixer
  2. Direct plain UDP inputs can also feed AISMixer
  3. Tag parsing (c/g) and multipart reassembly
  4. Deduplication + meta cleaning (strip vendor headers)
  5. Optional meta add/normalize
  6. Fan‑out forwarding (per user / per target)
  1. AIS приемник/приложение → локален UDP → nmea_sproxy → криптиран SEC вход → AISMixer
  2. Директни plain UDP входове също могат да подават към AISMixer
  3. Парсинг на тагове (c/g) и сглобяване на multipart
  4. Дедупликация + почистване на мета хедъри
  5. Добавяне/нормализиране на мета по избор
  6. Форуърд към няколко цели (по потребител / дестинация)

Focused roles

Ясни роли

AISMixer is the mixer, deduplicator, tag‑aware normalizer, and fan‑out forwarder.

AISMixer е миксерът, дедупликаторът, tag‑aware нормализаторът и fan‑out форурдърът.

nmea_sproxy is a client‑side secure shovel/proxy: one process maps one local UDP input to one encrypted AISMixer SEC input. It does not mix streams.

nmea_sproxy е клиентско secure shovel/proxy: един процес свързва един локален UDP вход с един криптиран AISMixer SEC вход. То не смесва потоци.

Get started

Първи стъпки

Run AISMixer directly, run one manual nmea_sproxy relation, or start an installed singleton/template systemd service:

Стартирай AISMixer директно, пусни една ръчна nmea_sproxy връзка или стартирай инсталирана singleton/template systemd услуга:

$ git clone https://github.com/iliyan85/aismixer
$ cd aismixer
$ python3 aismixer.py
$ cd nmea_sproxy && python3 nmea_sproxy.py
$ sudo systemctl start nmea_sproxy
$ sudo systemctl start nmea_sproxy@boat

Template instance names such as boat, yacht, or balchik_roof are operator‑chosen labels. Each instance config defines its own one‑input‑to‑one‑secure‑output relation.

Template instance имена като boat, yacht или balchik_roof са етикети, избрани от оператора. Конфигурацията на всяка инстанция описва собствена връзка от един вход към един защитен изход.
